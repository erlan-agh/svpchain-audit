🛡️ SVP Chain — Authorized Audit (Testnet) Findings
by Nalreee | repo + PoC: github.com/erlan-agh/svpchain-audit

Scope: svpchain/evm, oracle, mcp, agent, bridge UIs, SVPBridge, Lendora, MCP server.
✅ No Critical/High — code is well-hardened.

🔸 #1 CORS `*` on /api/health (pre-bridge.svpstars.com) — Info/Low
Header on /api/health + 404s only; body is {"result":"ok"}, no sensitive data. Fix: scope CORS to allowlist.

🔸 #2 SVPBridge sig-loop order-dependent false-negative — Low/Med
SVPBridge.sol L898–918 advances sigIdx only on match, so mis-ordered sigs can fail quorum despite >2/3 power present → legit withdrawals rejected (liveness/DoS). Dup sig does NOT double-count. Fix: sort sigs or map signer→power. Repro: poc_signature_loop.py.

🔸 #3 Hidden admin fn in Lendora prod — Medium
`watch_tg_invmru_ae5c248(uint256,bool,bool)` (sel 0x5407cedf) present in deployed bytecode (keccak-verified) + REVERTS for non-owner (access-controlled). Undocumented debug fn in fund-custody contract = hygiene risk if owner key compromised. Repro: poc_lendora_hidden_fns.py. (5 other "watch_tg_*" labels from 4byte.directory were pranks = standard Compound fns; only this one real.)

🔸 #4 Lendora/NovaSwap UNVERIFIED — Medium (gap)
$50K lending/perps scope unauditable (Cosmos-EVM bridge; EVM storage empty). Request source verification before mainnet.

🔸 #5 Staging `pre-*` = prod-identical — Low
pre-lendora/pre-swap/etc serve same apps + addrs. Pivot risk if staging has weaker auth. Restrict staging.

🔸 #6 MCP auth gate — SOLID (pass)
Legit session via keccak256(R1||chainId||nonce||exp) 65-byte sig; cross-tenant get_subaccount(B) under A → "owner B not allowed for tenant". No IDOR. Repro: mcp_idor.py.

📝 All findings = hardening/liveness/transparency, not theft. Mainnet (2518) not live yet — re-scan on launch.
